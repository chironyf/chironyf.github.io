排序成本模型

  • 在研究排序算法时，我们需要计算比较和交换的数量。对于不交换元素的算法，我们会计算访问数组的次数

测试代码

  • 对于某个排序算法，需要检验排序的结果正确与否，往往需要编写测试代码。本文中，我们将根据排序之后的元素是否有序来验证，例如按升序排列，只需要遍历数组，如果存在$a[i] > a[i+1]$则排序失败。但这种方法只是一种表象的检验，排序算法有没有更改原数组是不知道的，例如$5\ 4\ 3\ 2\ 1$给排序成了$0\ 0\ 0\ 0\ 0$，结果也会显示排序成功。由于目前我还没找到合适的测试代码，姑且用这个吧。

选择排序

外层循环从数组的第$1$个元素遍历到数组的最后一个元素，每遍历到$i$位置，就将$i$当成是该位置最小元素的下标，内循环从$i+1$开始，遍历到数组最后一个元素，筛选出真正最小元素的下标$min$，然后将$a[i]$与$a[min]$交换即可。

  • 每次交换都能排好一个元素，故总共只需要$N$次交换就能排序完毕。在内循环寻找当前最值的时候，保存的是当前最值的下标，而不是最值本身。由于最值下标的更新代价可能比直接交换数组元素的代价小得多，因此忽略不计。
  • 对于数组的最后一个元素，进行了$0$次比较，自己和自己交换了$1$次。对于数组中的从$0\ldots{N-1}$的任意一个位置$i$的确定，都需要${N-1-i}$次比较。因此总共比较次数为：$$\sum_{i=0}^{N-1}{N-1-i} = \frac{N(N-1)}{2}$$这种排序的运行时间总是不变的，不管输入是随机还是有序，都需要$N$次交换以及$\frac{N(N-1)}{2}$次比较。
  • 交换次数$N$和数组的大小是线性关系，其它任何算法都达不到这个要求。

插入排序

外层循环从数组的第$2$个元素遍历到最后一个元素，每遍历到$i$位置，内层循环就从该位置（$j = i$）开始往前遍历（注意边界），每当发现$a[j] < a[j - 1]$时，就交换这两个元素。

  • 当数组有序时达到排序最好情况，对于从$1\ldots{N-1}$的每个元素只需要和前一个元素比较就能确定自己的位置，由于已经有序，故无需交换。因此，最好情况下，只需要$N-1$次比较和$0$次交换即可完成排序。
  • 当数组逆序时达到排序最坏情况，对于从$1\ldots{N-1}$的每个位置$i$元素，都需要进行$i$次比较以及$i$次交换，次数为$$\sum_{i=1}^{N-1}{i} = \frac{N(N-1)}{2}$$即最坏情况需要～$\frac{N^2}{2}$次比较和交换
  • 平均情况下，需要～$\frac{N^2}{4}$次比较和～$\frac{N^2}{4}$次交换。
  • 比较次数与交换次数的关系：交换一次的前提必定是比较了一次，但是比较了却不一定会交换。当元素恰好处于正确位置时，只是比较了一次，并没有交换。因此，比较次数$compTimes$与交换次数$exchTimes$的关系是：$$comTimes = exchTimes + N - extraTimes$$其中$extraTimes$是当前元素正好在该位置的次数。
  • 在部分有序的情况下（倒置的数量很少时），比其他任何排序算法都快。
  • 改进：在内循环中将较大的元素都向右移动而不总是交换两个元素，这样能使访问数组的次数减半。

希尔排序

避免了插入排序中的元素一个一个挪动造成了效率低下问题，实现了元素的远距离交换，是基于插入排序的一种改进算法。最外层是一个缩短步长的大循环，内层的两个循环与插入排序同理（姑且叫做中层循环和内层循环），中层循环从$h-$有序子数组的第二个元素开始，依次遍历到$N-1$，内层循环实现元素的交换。

  • 由于子数组众多，故存在众多的第二个元素，为$a[h]\ldots{a[2h - 1]}$。
  • 子数组的插入排序是同时进行的，并不是说第一个子数组排完后再排第二个，而是第一个子数组的第一个元素插入成功后紧接着对第二个子数组的第一个元素进行插入操作。
  • 性质：使用递增序列$1, 4, 13, 40, 121, 364\ldots$（$h = 3h + 1$）的希尔排序所需的比较次数不会超出N的若干倍乘以递增序列的长度。
  • 本排序算法的性能有待研究

归并排序

是分治思想的一种应用。
对于自顶向下的归并排序来说，先将原数组分解成只有$1$个元素的数组（一个元素认为是有序的），然后合并成有序的完整的数组，主要方法利用递归实现。代码中的归并部分（$merge$）利用的是原地归并的方法，即数组$a[lo\ldots{mid]}$和$a[mid+1\ldots{hi]}$借助于与其同样大小的数组将前后两个有序的部分合并成一个有序部分存储在原数组中。
对于自底向上的归并排序来说，首先将数组的所有元素当作元素只有$1$个的子数组进行归并，然后是两两归并，四四归并…直到所有归并完毕，总共会进行$lgN$次归并。
下面的图片直观地展现了归并排序的基本过程（图片来自维基百科）

  • $N$个元素的数组归并排序时子数组依赖树的深度是$lgN$，第$i$层的节点数目为$2^i$，每个节点内部元素的数目为$\frac{N}{2^i} = N$，故每一层归并时最大可能的比较次数是$2^i\frac{N}{2^i}$，当归并的两个子数组某一个中的所有元素都归并完毕且另一个子数组还有元素没有归并完毕时，比较的次数会小于$N$，所以总共最大可能的比较次数为$NlgN$，归并排序需要$\frac{1}{2}NlogN$～$NlgN$次比较。
  • 归并排序最多需要访问数组$6NlgN$次，对于每一层来说，其中复制需要$2N$次（每复制一个元素需要访问两个数组），将排序好的元素移到原数组也需要$2N$次（同理），比较最多需要$2N$次。
  • 缺点在于需要额外的存储空间，并且大小与数组的大小成正比。
  • 自顶向下或者自底向上的方式在数组的长度为$2^N$时，比较次数和数组访问次数正好相等。
  • 自底向上的方法比较适合用链表组织的数据，链表的原地排序不需要创建任何新的结点。
  • 没有任何基于比较的算法能够保证使用少于$lg(N!)$~$NlgN$次比较将长度为$N$的数组排序。
