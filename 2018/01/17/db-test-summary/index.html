数据库期末复习知识点总结

Jan 17, 2018 | 阅读

总结一下数据库复习的一些知识点。

1. 基础概念

1.1 数据(data)

描述事物的符号

1.2 数据库(database, DB)

长期存储在计算机内、有组织、可共享的大量数据的集合
其中的数据有如下特点：

  • 有组织
  • 可共享
  • 冗余度小
  • 独立性高
  • 永久存储

1.3 数据库管理系统(DBMS)

位于用户与操作系统之间的一层数据管理软件
主要功能：

  • 数据定义
  • 数据组织、存储和管理
  • 数据操纵功能
  • 数据库的事务管理和运行管理
  • 数据库的建立和维护

1.4 数据库系统(DBS)

由DB, DBMS, 应用程序和数据库管理员(DBA)组成
特点：

  • 数据结构化
  • 数据共享性高、冗余度底且易扩充
  • 数据独立性高
  • 数据由DBMS统一管理和控制

1.5 数据模型（层次模型、网状模型、关系模型等）

对现实世界数据特征的抽象，是(DBS)的核心和基础
分类：

  • 概念模型
  • 逻辑模型和物理模型

组成要素：

  • 数据结构
  • 数据操作
  • 数据的完整性约束条件

1.6 DBS的三级模式与二级映像

三级模式：

  • 模式（一个）
  • 外模式（子模式、用户模式，多个）
  • 内模式（存储模式，一个）

二级映像

  • 外模式/模式映像
    模式改变，只需改变该映像，外模式不用变，体现了数据的逻辑独立性
  • 模式/内模式映像
    内模式改变，只需改变该映像，模式不用变，体现了数据的物理独立性

2. 关系数据库

接下来总结关系数据库的重要知识点

2.1 关系

三种类型：

  • 基本关系（基本表）
  • 查询表（查询结果对应的表）
  • 视图表（虚表）

2.2 关系模型

数据模型的一种，由以下元素组成：

  • 关系数据结构
  • 关系操作集合
  • 关系完整性约束

2.3 关系模式, R(U, D, DOM, F)

对关系的描述，包括：

  • 关系名, R
  • 属性名集合, U
  • 属性来自的域, D
  • 属性向域的映像集合, DOM
  • 属性间数据的依赖关系集合, F

2.4 关系操作

  • 常用操作
    • 查询(query)
    • 插入(insert)
    • 删除(delete)
    • 修改(update)
  • 基本操作
    • 选择
    • 投影
    • 并
    • 差
    • 笛卡尔积
  • 查询操作
    • 基本操作
    • 连接(自然连接与等值连接的区别在于，比较的分量必须是同名的属性组，并且去掉重复元组)
    • 除(某属性的象集包含另一关系的某属性所有取值集合)
    • 交

2.5 关系的完整性

  • 实体完整性
    主属性不能是NULL
  • 参照完整性
    可以为空，不为空是必须是另一个关系的主码
  • 用户定义的完整性
    例如CHECK, NOT NULL…

4. 关系数据理论

4.1 基础

  • 数据依赖
    • 函数依赖(关系模式中的两个不同的属性组X和Y，只要X相同，必定Y相同，称为Y函数依赖于X)
      有完全函数依赖F，部分函数依赖P，传递函数依赖
    • 多值依赖
  • 问题
    • 数据冗余
    • 更新异常
    • 插入异常
    • 删除异常

4.2 范式

5NF ⊂ 4NF ⊂ BCNF ⊂ 3NF ⊂ 2NF ⊂ 1NF

  • 第一范式(1NF)，每一个分量都是不可分的数据项

从 1NF -> 2NF 消除非主属性对码的部分函数依赖

  • 第二范式(2NF)，在第一范式的基础上，每一个非主属性都完全函数依赖于候选码，没有部分依赖

从 2NF -> 3NF 消除非主属性对码的传递函数依赖

  • 第三范式(3NF)，在第一范式的基础上，没有属性对码是传递依赖的

从 3NF -> BCNF 消除非主属性对码的部分和传递函数依赖

  • BCNF
    • 所有非主属性对每一个码都是完全函数依赖
    • 所有主属性对每一个不包含它的码也是完全函数依赖
    • 没有任何属性完全函数依赖于非码的任何一组属性

从 BCNF -> 4NF 消除非平凡且非函数依赖的多值依赖
